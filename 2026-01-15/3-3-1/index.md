# 🔍 공개 API 응답 분석 실습 과제

---

## 📚 목표

> 공개 API의 응답 데이터를 확인하고 JSON 구조를 분석할 수 있습니다.

---

## 1. 공개 API URL 직접 접속하기

무료 공개 API인 JSONPlaceholder를 사용
기본 주소는 **https://jsonplaceholder.typicode.com** 이며,
게시글 목록을 확인하기 위해 **/posts 엔드포인트**에 접속.

브라우저 주소창에 아래 URL을 직접 입력하여 JSON 응답을 확인.
https://jsonplaceholder.typicode.com/posts

---

## 2. JSON 응답 구조 분석

posts API의 응답은 여러 개의 게시글 객체가 담긴 배열 형태
각 게시글 객체는 아래와 같은 JSON 구조를 가진다.

```json
{
  "userId": 1,
  "id": 1,
  "title": "첫 번째 글",
  "body": "내용..."
}
```

| 필드명 | 타입   | 의미                      |
| ------ | ------ | ------------------------- |
| userId | Number | 게시글을 작성한 사용자 ID |
| id     | Number | 게시글 고유 식별자        |
| title  | String | 게시글 제목               |
| body   | String | 게시글 내용               |

→ 이 구조를 통해서 누가 어떤 글을 어떤 제목과 내용으로 작성했는지 알수 있다.

---

## 3 Request와 Response 차이 설명

- ## 📤 Request (요청)

"게시글 목록 주세요"
방향 : **클라이언트 -> 서버**
데이터를 요청하는 것
ex : "게시글 목록 주세요"

> → /posts URL을 입력하는 것도 Request에 해당한다.

| 구분   | 내용                           |
| ------ | ------------------------------ |
| 메서드 | POST                           |
| URL    | /users                         |
| 헤더   | Content-Type: application/json |

- ## Response (응답)

방향 : **서버 -> 클라이언트**
요청에 대한 결과를 반환하는 것
es : "여기 게시글 10개요"

> → 서버는 **데이터를 JDON형식**으로 반환한다.

| 상태 코드   | 의미             |
| ----------- | ---------------- |
| 201 Created | 사용자 생성 성공 |

> **같은 URL이여도 메서드가 다르면 다른 엔드포인트이다**

---

## 4. API 키 필요 이유 및 보안 주의사항

- **API 키가 필요한 이유**

  - 사용자 및 서비스 식별
  - 무단 사용 방지
  - 요청 횟수 제한 및 트래픽 관리

- **보안 주의사항**
  - API 키는 비밀번호처럼 관리
  - GitHub에 절대 업로드 ❌
  - 환경변수(.env)를 사용해 안전하게 저장
  - 코드에 직접 작성하지 않는다
