# 🔍 상태 코드별 응답 시뮬레이션하기

---

## 200 400 500 각 상태코드 이론과 차이

## 2XX

> 요청이 성공적으로 처리되었음을 알리는 코드

1. **200 OK** - 가장 일반 적인 성공

2. **201 Created** - 새 리소스 생성 (POST 요청으로 새 데이터가 만들어진다)

3. 204 No Cortent - 성공, 응답 데이터 없음 (DELETE 요청 등에서 사용)

## 4XX

> **클라이언트(요청)에 문제가 있음을 나타내는 코드**

1. 400 Bad Request - 잘못된 요청 (요청 형식이 잘못됨)
   - JSON 형식 오류
   - 필수 파라미터 누락
   - 잘못된 데이터 타입

2. 401 Unauthorized - 인증 필요 (로그인이 필요한 요청)

   > 당신이 누군지 모르겠어요
   - 토큰 없음
   - 토큰 말료
   - 잘못된 인증 정보

3. 403 Forbidden - 권한 없음 (인증은 됐지만 권한이 부족)
   > 당신이 누군지는 알지만, 권한이 없어요

- 관리자 전용 페이지
  - 다른 사용자의 데이터
  - 접근 차단된 리소스

4. 404 Not Found - 리소스 없음 (요청한 것이 존재하지 않음)
   - 잘못된 URL
   - 이미 삭제되 페이지
   - 오타가 있는 경로

## 5XX

> 서버에 문제가 있음을 나타낸다

1. 500 Internal Server Error - 서버 내부 에러 (서버에서 예상치 못한 오류발생)
   - 코드 버그
   - 예외 처리 안 된 에러
   - 서버 설정 문제

2. 502 Bad Gateway - 게이트웨이 에러 (중간 서버가 응답을 받지 못함)
   - 프록시 서버 문제
   - 백엔드 서버 다운
   - 네트워크 연결 문제

3. 503 Service Unavailable - 서비스 불가 (서버가 일시적 사용불가)
   - 서버 과부화
   - 유지보수 중
   - 일시적 장애

4. 504 Geteway Timeout - 게이트웨이 타임아웃 (서버가 응답시간 초가)
   - 처리 시간이 너무 오래 걸림
   - 백엔드 서버 무응답
   - 네트워크 지연

---

## 4XX - 5XX 차이

- 에러의 원인이 클라이언트인지 서버인지의 차이로 구분 한다
  > - **4XX = 클라이언트 에러 , 5XX = 서버 에러**
  > - 4XX는 클라이언트의 잘못된 요청으로 발생하며 5XX는 요청은 정상이지만, 서버 내부에 문제로 발생한다.

**표로 구분하기**
| 구분 | 400 | 500 |
|---|---|---|
| 오류 유형 | 클라이언트 오류 | 서버 오류 |
| 원인 | 잘못된 요청 | 서버 내부 문제 |
| 요청 수정 | 필요 | 불 필요 |
| 책임 | 사용자 | 서버 |
